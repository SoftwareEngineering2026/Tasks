# 106. CI/CD

## Цель работы

Настроить автоматический редеплой проекта при изменении кода.

## Описание работы

В рамках лабораторной работы необходимо настроить CI/CD-процесс для программного проекта, организовать:

- автоматический запуск pipeline при изменении кода в репозитории;
- установку зависимостей проекта;
- запуск сборки проекта;
- запуск тестов или простой команды проверки работоспособности;
- получение статуса успешного или неуспешного выполнения pipeline.

В качестве проекта можно использовать:

- простое приложение из предыдущих лабораторных работ;
- собственный небольшой проект;
- любой open-source проект;

В качестве платформы CI/CD необходимо использовать один из инструментов:

- **GitHub Actions**;
- **GitLab CI/CD**;
- **Jenkins**.

Необходимо показать, что pipeline:

- автоматически запускается при `push` в репозиторий;
- выполняет заданные шаги последовательно;
- завершается успешно при корректном проекте;
- завершается с ошибкой при наличии проблемы в проекте.
