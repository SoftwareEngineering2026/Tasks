# 107. Monitoring

## Цель работы

Изучить подходы к централизованному сбору логов и мониторингу параметров микросервисного приложения.

## Описание работы

В рамках лабораторной работы необходимо:
- Развернуть микросервисное приложение, 
  Микросервисное приложение можно использовать любое, в том числе из лабораторной Practice105 (своё или любое из репозитория [Practice105](https://github.com/SoftwareEngineering2026/Practice105)).
- Реализовать собственную прикладную метрику (например, число запросов от пользователей) с публикацией через endpoint `/metrics`
- - организовать визуализацию логов и этой метрики в дашборде Grafana. 


В качестве инструментов рекомендуется использовать:

- **Grafana Alloy** — для сбора логов и метрик;
- **Loki** — для хранения логов;
- **Grafana Mimir** — для хранения метрик;
- **Grafana** — для визуализации метрик и логов.
