# 106. Continuous Integration (CI) 

## Цель работы

Настроить процесс автоматического тестирования входящих изменений при pull request в главную ветку.

## Описание работы

В рамках лабораторной работы необходимо настроить CI-процесс для программного проекта, организовать:

- автоматический запуск pipeline при pull request в главную ветку репозитория;
- установку зависимостей проекта;
- запуск сборки проекта;
- запуск тестов;

В качестве проекта можно использовать:

- простое приложение из предыдущих лабораторных работ;
- собственный проект;
- любой open-source проект;

В качестве платформы CI/CD использовать один из популярных инструментов CI на рынке. Например:

- **GitHub Actions**;
- **GitLab CI/CD**;
- **Jenkins**.

Необходимо показать, что pipeline:

- автоматически запускается при `pull request` в главную ветку;
- выполняет заданные шаги последовательно;
- завершается успешно при корректном проекте;
- завершается с ошибкой при наличии проблемы в проекте.
