# 103. Refactoring

## Цель

Получить практический опыт анализа и рефакторинга существующего программного кода.

---

# Задание

1. Сделать **fork** проекта:  
   <https://github.com/SoftwareEngineering2026/103-refactoring-exercise>

   Или выбрать **собственный / open-source проект** по желанию.

2. Склонировать репозиторий, выстроить локальное окружение, запустить код.

3. Проанализировать код проекта и выбрать участок, требующий рефакторинга.

4. Провести **рефакторинг выбранного участка**.

5. Проверить, что:
   - код корректно работает после изменений
   - участок рефакторинга **покрыт тестами**

6. Сделать **Pull Request** владельцу проекта.

---

# Требования к Pull Request

Pull Request должен соответствовать следующим требованиям:

- изменения оформлены в **отдельной ветке**
- ветка находится в **healthy состоянии** (проект собирается, тесты проходят)
- коммиты **атомарные** (каждый коммит содержит логически завершённое изменение)
- изменения **покрыты тестами**
- код после рефакторинга сохраняет исходную функциональность

---

# Требования к отчёту

В репозитории  
https://github.com/SoftwareEngineering2026/Practice103

необходимо создать папку:

`ФамилияИО/`


Внутри папки разместить файл:

`ФамилияИО.md`

Структура отчёта свободная.

---

# Что указать в отчёте

1. **Выбранный проект (если выбрали свой или os репозиторий)**
   - ссылка на репозиторий
   - краткое описание проекта
   - стек технологий

2. **Краткое описание использованных принципов рефакторинга**

3. **Описание выполненного рефакторинга**
   - какие проблемы были обнаружены
   - как эти проблемы решились

4. **Ссылка на созданный Pull Request**
